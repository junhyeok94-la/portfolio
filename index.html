MAJOR CASE

GS Retail HBU
데이터플랫폼 최적화

2025.07 - 현재 (5년차)

AWS MWAA Redshift dbt EMR

Problem (Why)

기존 배치 시스템은 단순 시간 기반(Time-based) 스케줄링으로 인해 상위 작업 완료와 무관하게 리소스를 대기하거나 데이터 정합성 이슈가 발생했습니다. 또한 메타 DB 부하로 인해 전체 스케줄링이 지연되는 병목 현상이 있었습니다.

Action (How)

ExternalTaskSensor 기반의 **이벤트 드리븐 구조**로 전면 개편하고, Deferrable Operator를 적용하여 워커 리소스 점유를 최소화했습니다. 백엔드 DB 클렌징 DAG를 직접 구축하여 인프라 성능을 상시 최적화했습니다.

Key Result

배치 종료 시간 11:00 → 05:30 (5.5시간 조기 마감)

데이터 플랫폼 구축

2023.09 - 2024.12

TeraStream Java Oracle

Problem

4개 사법기관의 파편화된 데이터 형식과 BLOB/CLOB 등 대용량 비정형 데이터 전송 시 잦은 에러로 인해 데이터 통합의 신뢰도가 낮았습니다.

Impact

Strategic Troubleshooting

5년 차 엔지니어의 차별점은 에러를 대하는 태도에 있습니다. 문제의 근본 원인을 파악하고 구조적으로 해결한 로그입니다.

#MWAA_Stability

메인터넌스 시 태스크 중단 이슈

MWAA 업데이트 시 모든 워커가 재시작되며 실행 중인 태스크가 강제 종료되는 현상 발생.

// Solution: Deferrable Operator
control_dag(deferrable=True)
sub_dag(retry_exponential_backoff)

결과: 업데이트 기간 중 데이터 누락률 0% 달성

#DB_Optimization

Meta-DB CPU 사용량 급증

장기 가동 시 `rendered_task_instance_fields` 테이블 부하로 스케줄링 큐 지연 발생.

// Action: DB Cleanse Strategy
delete_old_records_daily()
vacuum_analyze_airflow_db()

결과: 스케줄링 대기 시간 85% 단축

#Data_Integrity

배치 간 무결성 검증 누락

적재 완료 후 중복 데이터가 적재되어 대시보드 지표가 왜곡되는 비즈니스 손실 발생.

// Design: Double-Check DAG
last_task_success() -> trigger_qc()
if duplicate: alert_and_stop()
