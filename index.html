배치 최적화 매니저

AWS MWAA Redshift dbt Grafana

기존 레거시(Oracle SAS) 환경의 홈쇼핑 데이터를 Airflow, dbt, Redshift 기반의 Modern Data Stack으로 전환하는 프로젝트입니다. 프로젝트 중반에 투입되어 배치 목표 시간 달성, 비용 절감, 코드 표준화를 주도했습니다.

1. Redshift Provisioned + Serverless 라우팅 비용/성능 최적화

  • 문제: DAG 내 모든 dbt 모델을 Serverless로 돌리면 비용이 폭증하고, Provisioned로 돌리면 무거운 모델로 인해 배치가 지연되는 트레이드오프 발생.
  • 해결/성과: dbt Task 부하를 모니터링하여 가벼운 작업은 Provisioned에서, 무거운 특정 Task만 Serverless로 동적 라우팅되도록 Airflow 분기 로직 개발. 무리한 스케일업 없이 배치 시간을 달성하고 불필요한 과금을 방어함.

2. Airflow Variable 기반 Stateful 소급(Backfill) 아키텍처

  • 문제: 기존 시스템은 Hourly 배치 시 최대 4시간까지만 소급 처리가 가능하여 데이터 복구에 제약 존재.
  • 해결/성과: Airflow Variable을 활용해 DAG의 성공 기준 data_interval_end를 키(Key)로 추적하는 상태 관리 로직 도입. Hourly 배치 소급 가능 시간을 24시간으로 대폭 확장하여 운영 안정성 확보.

3. 파이프라인 코드 클렌징 및 Naming/Tagging 표준화

  • 해결: 미사용/중복 코드를 전면 클렌징하고 모듈화. Control DAG(Trigger 수행)와 Sub DAG의 네이밍/태그 표준을 수립(예: 메인 DHUB=1, 외부=2).
  • 성과: 수백 개의 파이프라인 목적과 레이어를 직관적으로 파악할 수 있도록 체계를 확립하여 팀 전체 생산성 향상.

4. SAS to dbt(ELT) 전환 시 데이터 정합성 보장

  • 문제: 제약 조건을 강제로 넘기는 기존 SAS 관행(force)으로 인해 Redshift 전환 시 잦은 적재 에러 발생.
  • 해결/성과: SAS의 Temp 테이블 로직을 dbt 워킹 테이블(WK_) 구조로 매핑. 원천 데이터 도메인 길이를 선제적으로 재조사하여 기존의 데이터 유실(Data Loss) 문제를 해결하고 안정적인 기반 스키마를 확립.

5. Grafana 모니터링 재구축 및 메타 DB 부하(Slow Query) 해결

  • 문제: 기존 Grafana 대시보드가 Airflow 메타 DB(MySQL)를 직접 참조하며 스케줄러 지연 및 Lock 경합 발생.
  • 해결/성과: 기존 로직 전면 폐기 및 인덱스를 고려한 경량화 쿼리로 대시보드 재구축. 시각적 모니터링을 고도화함과 동시에 Airflow 코어 시스템의 응답 속도를 정상화함.

OLAP 개발 선임 & Admin

BI-Matrix InnoQuartz SAP Hana

행정안전부 및 전국 지자체가 사용하는 정책지원시스템의 대규모 통계/BI 화면 구축 및 이기종 데이터 연계 파이프라인을 개발했습니다.

1. 대규모 OLAP 화면 구축 및 'One-Table' 전략 도입

화면별로 개별 물리 테이블을 생성하는 기존 방식은 유지보수 공수를 기하급수적으로 늘릴 우려가 있었습니다. 화면 ID를 파티션 키로 사용하는 통합 테이블(One-Table) 모델을 설계하여, 할당된 범위를 초과하는 200여 개의 통계 화면 개발을 기한 내에 안정적으로 완수했습니다.

2. 고난이도 행정 비즈니스 로직 전산화 (유형분류 및 위기관리)

매년 외부 용역에 의존하던 '자치단체 유형분류'(인구 50%, 재정 50% 가중치 평가) 및 '재정위기관리' 작업을 시스템화했습니다. 행안부 공무원과 직접 요구사항을 조율하고, 외부 Open API 데이터를 융합해 복잡한 지표 산출 로직을 직접 파이프라인에 구현하여 30여 본의 심화 화면을 오픈했습니다.

3. OLAP Admin 및 통합 수집 파이프라인 구축

InnoQuartz를 활용해 통계청/한국은행 Open API 데이터를 수집하는 ETL을 구축했습니다. 프로젝트 종료 시점에는 OLAP Admin으로서 아키텍처 및 화면 인수인계 문서를 완벽히 작성하여 안정적인 이관을 주도했습니다.
