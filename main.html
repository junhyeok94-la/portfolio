Problem Definition

기존 배치가 상위 작업의 완료 여부와 무관한 단순 시간 기반(Time-based)으로 동작하여, 데이터 누락 위험이 존재하고 불필요한 리소스 대기 시간이 발생했습니다. 또한, MWAA 메타 DB 로그 누적으로 인해 CPU 사용량이 급증하며 스케줄링 지연이 빈번했습니다.

Solution Strategy

  • Event-Driven: ExternalTaskSensor 도입으로 Hourly/Daily 유기적 연동.
  • Stability: deferrable=True 옵션 적용으로 워커 점유 최소화.
  • Maintenance: DB 클렌징 DAG 및 중복 체크 DAG 구축.
